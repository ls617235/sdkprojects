## 二、接口清单

### 1. AI 对话接口（核心）

#### 1.1 流式对话接口

```typescript
POST /api/app_chat?stream=true
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| app_info | string | 是 | 应用标识，固定为 `digital_intelligent_audit` |
| input_value | string | 是 | 用户输入内容 |
| session_id | string | 否 | 会话ID，用于保持对话上下文 |
| tweaks | object | 否 | 流程参数调整 |

**请求示例**:

```json
{
  "app_info": "digital_intelligent_audit",
  "input_value": "请分析这份审计文件",
  "session_id": "session_1234567890",
  "tweaks": {}
}
```

**响应格式**（NDJSON 流式响应）:

```
{"event": "token", "data": {"chunk": "一、客户基本信息"}}
{"event": "token", "data": {"chunk": "\n\n户名：张三"}}
...
{"event": "end", "data": {"result": {...}}}
```

**事件类型**:

| 事件名 | 说明 |
|--------|------|
| token | 流式文本片段 |
| end | 响应结束 |
| error | 发生错误 |

---

#### 1.2 非流式对话接口

```typescript
POST /api/app_chat?stream=false
```

**请求参数**: 同上

**响应格式**（JSON）:

```json
{
  "success": true,
  "data": "完整的AI回复内容",
  "raw": {...}
}
```

```
                                                                    ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  显示在界面  │◀────│  保存AI回复  │◀────│  流式接收响应  │◀────│  后端处理   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 3.2 流式响应处理流程

```typescript
// 1. 发起流式请求
const response = await appChatByAppInfoAdaptive(
  'digital_intelligent_audit',
  inputText,
  {},
  {
    onToken: (chunk: string) => {
      // 2. 实时更新流式消息
      setCurrentStreamMessage((prev) => prev + chunk);
    },
    onComplete: (fullText: string) => {
      // 3. 完成后保存完整消息
      streamCollected = fullText;
    },
  },
  sessionId,
  true  // 启用流式
);
```

## 四、数据类型定义

### 4.1 聊天消息

```typescript
interface ChatMessage {
  id: string;           // 消息唯一标识
  text: string;         // 消息内容
  sender: 'user' | 'ai'; // 发送者类型
  timestamp: string;    // ISO时间戳
}
```

### 4.2 会话信息

```typescript
interface ConversationItem {
  id: string;
  conversation_name: string;
  created_at: string;
  user_id: number;
  user_name: string;
  app_info: string;
  message_count?: number;
  last_message?: string;
}
```

### 4.3 历史消息

```typescript
interface HistoryMessage {
  id: string;
  conversation_id: string;
  content: string;
  role: 'user' | 'assistant';
  created_at: string;
  updated_at: string;
}
```

### 4.4 流式响应

```typescript
interface StreamResponse {
  event: 'token' | 'end' | 'error';
  data: {
    chunk?: string;      // token事件的文本片段
    text?: string;       // end事件的完整文本
    result?: object;     // end事件的完整结果
    message?: string;    // error事件的错误信息
  };
}
```

## 五、错误处理

### 5.1 错误码说明

| HTTP状态码 | 说明 |
|-----------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权，Token无效 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.2 前端错误处理

```typescript
try {
  const response = await appChatByAppInfoAdaptive(...);
} catch (error) {
  console.error('发送消息失败:', error);
  message.error('发送消息失败，请重试');
}
```
